# Weight presets

## What
One-click named presets for the "Pondération des sources" sliders, alongside
the existing free-form per-source sliders — not a replacement for them.

## Why
`RecoWeightsSection` currently exposes 9 independent sliders
(`DEFAULT_WEIGHTS` in [src/lib/recoWeights.ts](../../src/lib/recoWeights.ts)),
each committed individually to the URL's packed `w` param. Finding a
meaningfully different "mode" (e.g. lean harder on crowd consensus vs. lean
into niche taste-matching) currently means manually dragging 3-4 sliders and
remembering what combination worked. The main anime list already solves this
exact problem for filters via `VIEW_PRESETS`
([src/lib/animeUrlParams.ts:497](../../src/lib/animeUrlParams.ts)) — presets
that set several fields at once, then let the user fine-tune from there.

## Rough approach
- Define a small `RECO_WEIGHT_PRESETS` table in `recoWeights.ts` (client-safe,
  same file as `DEFAULT_WEIGHTS`), each entry a `Partial<SourceWeights>` plus a
  label, e.g.:
  - **Sûr** — crowd 1.4, suggestions 0.5, rejection/popularity untouched
  - **Découverte** — genre/studio boosted, popularity pushed more negative,
    crowd reduced
  - **Défaut** — resets to `DEFAULT_WEIGHTS`
- Applying a preset replaces the full weight map (like `applyPreset` does for
  filters) and commits once to the URL — the sliders underneath just reflect
  whatever the resulting `state.weights` is, so they stay usable for further
  manual tweaks after picking a preset.
- No server changes: this is purely a client-side convenience over the
  already-existing `w` URL param and `computeFeed(options.weights)` path.

## Touch points
`src/lib/recoWeights.ts`, `RecoWeightsSection.tsx`, no API/lib changes.
