# Freshness badge

## What
A small "Nouveau" badge on feed cards that entered the candidate set since
the previous refresh, so returning to the page after a refresh makes it
obvious what actually changed.

## Why
`RecommendationsData.lastRefresh`
([src/lib/recommendations.ts:155](../../src/lib/recommendations.ts)) is
already stored and shown in the sidebar, but only as a single timestamp for
the whole feed. Because ranking is recomputed live from stored edges on
every visit (ranking is cheap, fetch is expensive — see the file's top
comment), the same candidate can silently shuffle position across visits
even without a refresh, and after a refresh there's no way to tell "these 6
are genuinely new" from "these just re-ranked." For a feed you're expected
to check repeatedly over time, that distinction is the main thing worth
flagging.

## Rough approach
- Requires knowing candidate set membership *before* the most recent
  refresh, which isn't currently persisted (only the current edges are
  stored, previous ones are overwritten in `performRecommendationsRefresh`).
- Cheapest option: on `saveRecommendationsData`, snapshot the *previous*
  candidate id set into a small `previousCandidateIds: number[]` field on
  `RecommendationsData` before overwriting `seeds`/`suggestions`. `computeFeed`
  then marks `recoMeta.isNew = !previousCandidateIds.includes(id)` for each
  eligible item.
- Alternative without a schema change: derive "new" from `AnimeForDisplay`'s
  own `created_at`/`updated_at` MAL fields relative to `lastRefresh` — cheaper
  but conflates "new to MAL" with "new to *your* feed," which is a different
  (weaker) signal than intended here. Worth calling out as the fallback if
  the snapshot field is judged not worth the schema churn.
- Badge only needs to persist for one visit cycle — no need to track "seen"
  state per-card long-term, unlike the feedback store.

## Touch points
`src/lib/recommendations.ts` (`RecommendationsData` shape,
`performRecommendationsRefresh`, `computeFeed`), `RecoMeta` type in
`src/models/anime.ts`, card rendering in `AnimeCardView`.
