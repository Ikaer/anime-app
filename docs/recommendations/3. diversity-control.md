# Diversity control (re-rank for variety)

## What
A slider/toggle that re-ranks the top of the feed to avoid clustering — e.g.
capping how many consecutive (or top-N) cards can share a dominant studio or
genre — sitting on top of the existing affinity ranking.

## Why
`computeFeed` in
[src/lib/recommendations.ts:360](../../src/lib/recommendations.ts) sorts
purely by `Σ weight · normalizedSourceValue` (see the final `items.sort` at
line 556). That's a pure greedy ranking with no lookahead: if three seeds
all point hard at the same studio's back-catalog, or the taste profile
strongly favors one genre, the top of the feed can end up as a wall of
near-duplicates rather than a spread of options. The per-source weights
(`genre`, `studio`, ...) control *how much* those signals matter overall, but
nothing currently controls *distribution* across the ranked list itself.

## Rough approach
- Classic approach: Maximal Marginal Relevance (MMR)-style re-rank as a pass
  *after* `computeFeed` produces its scored `items` list, not a change to the
  scoring formula itself — keeps the additive weighted-sum model intact for
  the "Pourquoi ?" explain.
- Greedily build the output list: at each step, pick the next candidate that
  maximizes `affinityScore - λ · maxSimilarity(alreadyPicked)`, where
  similarity is a cheap overlap on the same `genre`/`studio` fields already
  extracted for `fieldMatch`. `λ = 0` reproduces today's exact ordering
  (backward compatible default).
- Expose as a single slider ("Ciblé ↔ Varié") mapped to `λ`, stored in the
  URL alongside the existing `w` param — one new numeric key, not a whole new
  section.
- Cost: O(n²) over the eligible candidate set per feed load; the eligible
  set here is small (bounded by crowd edges + suggestions, not the whole
  corpus), so this is cheap enough to run live like the rest of `computeFeed`.

## Touch points
`src/lib/recommendations.ts` (new re-rank pass after scoring),
`useRecommendationsUrlState.ts` (new `diversity` key), `RecoWeightsSection.tsx`
or a new small section for the single slider.
