# "More like this" drill-down

## What
A per-card action that opens a mini-feed of recommendations anchored on just
that one title, reusing the existing crowd/suggestions/taste-profile ranking
machinery instead of a new algorithm.

## Why
The feed today is a single global ranking across all seeds combined —
`computeFeed` accumulates affinity per candidate across every seed's edges
(the `perSeed` map in the `Accumulator` type,
[src/lib/recommendations.ts:351](../../src/lib/recommendations.ts)), and the
"Pourquoi ?" explain already surfaces `topSeeds` per card. But there's no way
to flip that around: "I like *this specific* recommendation, show me more
things like it specifically" — the per-card top-seeds are informational
only, not actionable.

## Rough approach
- The card already carries `recoMeta.topSeeds` (id + title + backers) and the
  full `recoMeta.breakdown`. A "Plus comme ça" action on a card doesn't need
  new fetch/edge data if the card itself has crowd edges already stored under
  `data.seeds` — but the common case is drilling down from an arbitrary
  candidate (not necessarily a seed), which has no edges of its own yet.
- Two viable shapes:
  1. **Cheap, no MAL call**: filter the *existing* stored `data.seeds` edges
     to only those the clicked title is a member of, i.e. treat it as the
     sole "seed" for a client-side re-filter of already-fetched data. Only
     works well if the clicked title happens to already be a candidate near
     other seeds — weak for a one-off arbitrary card.
  2. **Fresh, MAL call**: fetch `recommendations` for just that one anime id
     (`fetchRecoEdges`, already exists) and rank the result the same way,
     without touching the stored `RecommendationsData`. Slower (one MAL
     round-trip) but actually answers "more like this title" regardless of
     whether it was a seed.
  - Recommend (2): reuses `fetchRecoEdges` + a trimmed `computeFeed`-style
    scoring pass, exposed as a new lightweight endpoint (e.g.
    `GET /api/anime/recommendations/similar/[id]`) rather than mutating the
    stored feed.
- UI: open as a modal/side-panel list over the current card grid, not a full
  page navigation — keeps context of where you came from.

## Touch points
New `src/pages/api/anime/recommendations/similar/[id].ts`,
`src/lib/recommendations.ts` (extract a reusable single-seed scoring helper
from `computeFeed`), a new small modal component in `src/components/anime`.
